var Scope:
1.var declarations are function-scoped, meaning they are visible and accessible only within the function in which they are declared.
2.If a var variable is declared outside of any function, it becomes a global variable and is accessible throughout the entire program.
3.Hoisting: Variables declared with var are hoisted to the top of their scope. This means you can use a variable before it is declared in the code.
let Scope:
A block is a section of code that is bounded by. Curly braces house a block. A block is anything contained in curly brackets. Consequently, a variable declared in a block with let can only be used in that block.
const Scope:
Constant values are kept for variables specified using the const keyword. Let declarations and const declarations have some similarities. The scope of const declarations is blocked. Const declarations, like let declarations are only accessible within the block in which they were made. It is impossible to modify or re-declare const.

null
null is a special value in JavaScript that represents the intentional absence of any object value. It is often used when you want to explicitly set a variable to have no value or when an object property should be empty.
undefined
undefined is another special value in JavaScript that represents a variable that has not been assigned a value. When you declare a variable without assigning any value to it, JavaScript automatically assigns the value undefined.

REST API stands for Representational State Transfer Application Programming Interface. It is an architectural style for designing networked applications. In simpler terms, it's a set of constraints or principles that developers follow when creating web services. RESTful APIs are designed to be scalable, stateless, and provide a standard way of interacting with resources over the web.
